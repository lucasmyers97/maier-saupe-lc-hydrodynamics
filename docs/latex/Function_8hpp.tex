macro for declaring a new function that can be dynamically created.

This does two tasks for us.
\begin{DoxyEnumerate}
\item It defines a function to dynamically create an instance of the input class.
\item It registers the input class with the factory. This registration is done early in the program\textquotesingle{}s execution procedure because of the {\bfseries{attribute}} attached to it.
\end{DoxyEnumerate}

This macro should be called outside of all namespaces to avoid any clashing.

A good short answer to your questions about how {\bfseries{attribute}}\textquotesingle{}s work is \href{https://stackoverflow.com/questions/2053029/how-exactly-does-attribute-constructor-work}{\texttt{ https\+://stackoverflow.\+com/questions/2053029/how-\/exactly-\/does-\/attribute-\/constructor-\/work}} N\+O\+TE\+: {\bfseries{attribute}} is a function decorator supported by B\+O\+TH gcc and clang \href{https://gcc.gnu.org/onlinedocs/gcc/Function-Attributes.html}{\texttt{ https\+://gcc.\+gnu.\+org/onlinedocs/gcc/\+Function-\/\+Attributes.\+html}} \href{https://blog.timac.org/2016/0716-constructor-and-destructor-attributes/}{\texttt{ https\+://blog.\+timac.\+org/2016/0716-\/constructor-\/and-\/destructor-\/attributes/}} There are methods to do similar procedures that is portable to other compilers; however, their syntax is much more complicated.
