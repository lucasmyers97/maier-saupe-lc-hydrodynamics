This class does nothing except be the handle for all other types of objects in this \textquotesingle{}category\textquotesingle{}.

I think this is helpful because
\begin{DoxyEnumerate}

\item It avoid confusion with the factory design. With this typedef, getting an object would be functions\+::\+Factory\+::get().make(\char`\"{}foo\+::\+Bar\char`\"{}); While without, you would \mbox{\hyperlink{classfactory_1_1Factory_a632b97f5550412de9d3adf91e1ef33a6}{factory\+::\+Factory$<$\+Function\+Prototype$>$\+::get()}}.make(\char`\"{}foo\+::\+Bar\char`\"{}); even though one would be reasonable to assume to try \mbox{\hyperlink{classfactory_1_1Factory_a632b97f5550412de9d3adf91e1ef33a6}{Factory$<$foo\+::\+Bar$>$\+::get()}}.make(\char`\"{}foo\+::\+Bar\char`\"{}); which {\bfseries{would not work}}.
\end{DoxyEnumerate}

Moreover, notice that since we are in a different namespace that where Factory is defined, we can redefine the type \textquotesingle{}Factory\textquotesingle{} to be specifically for Function\+Prototypes in the functions namespace. This leads to the very eye-\/pleasing format functions\+::\+Factory\+::get().make(\char`\"{}foo\+::\+Bar\char`\"{});
