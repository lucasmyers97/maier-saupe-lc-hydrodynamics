macro for declaring a new function that can be dynamically created.

This does two tasks for us.

  1. It defines a function to dynamically create an instance of the input class.
  2. It registers the input class with the factory. This registration is done early in the program's execution procedure because of the attribute attached to it.

This macro should be called outside of all namespaces to avoid any clashing.

A good short answer to your questions about how attribute's work is https://stackoverflow.com/questions/2053029/how-exactly-does-attribute-constructor-work NOTE: attribute is a function decorator supported by BOTH gcc and clang https://gcc.gnu.org/onlinedocs/gcc/Function-Attributes.html https://blog.timac.org/2016/0716-constructor-and-destructor-attributes/ There are methods to do similar procedures that is portable to other compilers; however, their syntax is much more complicated.
